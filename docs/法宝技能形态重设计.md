# 法宝技能形态重构设计规范（最终版）

> 来源：多智能体工作流（审计复核 → 3 套设计哲学并行 → 综合）。日期 2026-06-18。
> 目标：拆解 arc/aura/nova/bolt 四大"万能筐"（原合计 42/67 件，约 63%），用最小代码/美术预算新增 6 个有辨识度的形态 + 7 条正交行为层 trait，让每个标签有专属招牌手感，玩家"关掉数字也能一眼分辨流派"。
> 所有形态复用现有 4 类投射物（straight/homing/orbit/glaive）与 4 类即时伤害（aoe/cone/chain/strike）及现有特效工厂，仅新增 1 个拉长几何 + 1 个绿色材质 + 1 条 sigil 轻量循环。**所有法宝 tags 一字未改，羁绊档位完全不变。**

| pierce | 穿透增强 | 提高直线弹命中数 | Projectile.pierceLeft 计数；剑系 |
| bounce | 弹射 | 命中改向最近敌继续飞 | Projectile.bounceLeft+nearestEnemyTo；雷系 |
| fork | 分叉 | 命中/击杀裂出 K=2 小弹 | 复用 spawnFission 思路抽通用；妖系 |

## 七、风险与缓解

1. **拆筐分布**：arc 14→0（改投 whirl/lance/forward/vine/bolt，arc 退役但保留 case 兜底）、aura 10→6（仅体系 5+妖白件 1）、nova 9→4（仅阵/体入门白蓝件）、bolt 9→9（妖系裂变招牌，加 fork/magnet 差异化）。
2. **性能（小游戏敏感）**：burn DoT 每帧只遍历 burnT>0 的敌人；sigil 每法宝只留最新 1 座防 aoe 叠满；vine 多发(count3)靠 spawn 早退兜底 maxProjectiles(260)。
3. **平衡**：lance/meteor 单发高伤配慢 CD；whirl 全圈击退复用现有衰减避免推出屏幕；burn 与魔系 berserk 叠加须 burnDps 封顶（≈base*0.4 且设绝对上限）；fork 与妖系 killExplode+lifesteal 三连须控 K=2、伤害×0.6。
4. **trait 签名扩散**：集中在 damageEnemy 末尾按透传字段写，即时形态用带 trait 的 aoe 包装。
5. **羁绊不变性**：67 件 tags 一字未改，tagCounts（按 id 去重）与 SYNERGIES 不受影响，每标签 need:9 仍可达。
6. **预览同步**：preview.ts 与 SKILL_FORM_DESC 必须同步补 6 个 case，否则图鉴预览空白。
